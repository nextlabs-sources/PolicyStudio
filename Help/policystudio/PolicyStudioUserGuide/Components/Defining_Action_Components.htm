Defining Action Components

The procedure for defining action components is quite different than for object components, since they are much more simply defined. In fact, they don’t have either properties or memberships at all, as object components do. Rather, each action component consists of a name, a description, and one or more basic actions that you can combine to meet your requirements.

Note: Actions appropriate for your configuration must be imported using the Seed Data procedure. For more information, consult the System Administrator User’s Guide.

With the exception of DAC actions (which should not be modified), you can cre­ate your own combinations of basic actions by creating new action components. To create a new action component, expand the Action Components panel to dis­play a list of all currently defined action components. Then perform the follow­ing steps:

  1. Click the New button, and type a name for the new action component. Since action components often combine more than one basic action, it is important to assign names that are as specific and descriptive as possible.

  2. Click OK to display the action editor controls.

  3. In the left pane, select the basic actions that you want to combine to define the new component. A component can consist of one basic action, or several.

  4. When you have made your selections, click the right arrow in the center to move the basic actions into the right-hand pane. If you make a mistake or change your mind, click the left arrow to remove basic actions from the component.

  5. In the Description field, provide a description of the action. This is helpful to remind yourself or other designers of the purpose of combining these basic actions into a single component.

  6. Click the Submit button.
