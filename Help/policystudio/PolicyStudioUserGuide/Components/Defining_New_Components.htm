Defining New Components

As discussed previously, you need to define components to represent various kinds of entities in your information environment. There are several times when you might want to define a new component:

·        After setting up your Control Center system, before constructing policies for the first time.

·        When new classes of information or users come under the control of infor­mation policy.

·        When a new policy requires a policy component that has not yet been cre­ated.

·        When conditions at the organization change in any way that adds new items to be covered by information control policies. For example, if the company reorganizes and adds a new division, you might need a new pol­icy component to represent the employees in that division.

When you are constructing a component, you do not need to save your work explicitly; it is automatically saved as you go. If you are interrupted while work­ing on a policy component, or you want to work on another task and return to constructing the policy component later, you can stop and continue the con­structing process as desired; your work will be saved in Draft status. You can find the policy component later in the appropriate component panel.

Component Types

As we have seen, there are the following categories of components:

·        Users

·        Applications

·        Computers

·        Actions

·        Devices

·        Documents

·        Portals

·        Servers

·        SAP

Note: An additional Resource component for SAP displays if you are designing policies for the Entitlement Manger for SAP ECC and SAP PLM. For more information about the SAP Resource component, refer to the User Guide for Entitlement Manager for SAP.

The components are organized into three panels in the lower left side of the Policy Author main window. Note that Users, Applications and Computers occupy separate tabs within the Subjects panel, while Devices, Documents, Portals, and Servers occupy separate tabs within the Resources panel.

All component types except Actions represent objects or network locations within your physical environment, and have basic structural and functional simi­larities. For this reason, we will discuss them together, under the general term object components. Actions are quite different, both in how they are defined and how they are used, and so we will discuss them separately.
