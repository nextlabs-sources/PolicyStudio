Management Tools

You can use Policy Author to monitor the current status of your deployed objects from several perspectives:

·        The component perspective: which policies incorporate a particular com­ponent (see Deployed Components)

·        The policy perspective: what is going on with all currently deployed poli­cies (see Deployed Policies)

·        The enforcer perspective: which policies each enforcer in your network is using, and how (see Deployments by Host)

·        The version history perspective: which version of each policy or compo­nent is currently deployed, and what other alternative versions exist (see Version History)

·        The audit logging perspective: view which users modified versions of poli­cies and when they did so (see Policy Audit Logging)

Deployed Components

One of the ways you can monitor current deployment is by component. That is, from time to time, you may need to find out which policies a particular compo­nent is being used in. To do this, select the component you are interested in, and select Actions > Show Policy Usage. Policy Author displays a list of all poli­cies that use the selected component.

You can also use the Search tool—the search field at the top of the policies tree—for this purpose. When you type a text string in the field and click Search, Policy Author displays a list of policies containing the string you typed. If the string is part of a component name, Policy Author display a list of policies con­taining that component. The search is not case-sensitive.

The example in the following figure shows the search results for the string “financial documents”. To return to the policy tree view, click the X button next to the Search button.

Deployed Policies

To view the current state of the policies you deployed, select Tool > Deploy­ment History. This displays a list of all policy deployments in the past eight days in chronological order, as shown in the figure below.

Note that this window is unrelated to any object you may have selected. It dis­plays information about recent deployments, regardless of what objects may be open or selected.

Each row in this list represents a single a deployment package. A deployment package simply refers to one policy and any associated components that were deployed together. (This is not required, but in practice, administrators gener­ally define several policies and then deploy them at the same time.) To see what policy and components were deployed in a particular package, double-click the row.

 

The Deployment History window

The table below describes the information presented in the Deployment History window.

Column

Description

Scheduled Time

The time at which this package was actually submitted for deployment, regardless of any specified delay before the scheduled deployment.

Deployment Time

The time at which this package is supposed to take effect. This may be the same as the Scheduled At time, or some point thereafter.

Contents

The number of items (policies + components) in this package.

Status

The current status of this package. There are four possible statuses:

•    Scheduled: The request for deployment has been accepted and a time has been set for it to occur. After scheduling a deployment, you can also Can­cel Scheduled Deployment; this returns the object to the Submitted state.

•    Deployed: Control Center has finished deploying the object.

•    Pending Deactivation: The request for deactivation has been accepted and a time has been set for it to occur.

•    Inactive: The object has been deactivated.

Canceling Deployment

You can use the Deployment History window to cancel deployment packages that are still in Scheduled state (see Cancelling). Note that the Cancel Selected Deployment button is not enabled unless a Scheduled item is selected in the list.

Deployments by Host

To check which policies and components are currently deployed on which Policy Enforcers in your network, select Tools > Deployment Status.

As the following figure shows, the Deployment Status window is organized into four tabs, for Desktop Enforcers, File Server Enforcers, Active Directory Enforc­ers, and Portal Enforcers. Each tab displays a list of all hosts—file servers or PCs—where policies have been deployed. For each machine in the Host list, the total numbers of policies and of policy components deployed are also displayed.

 

Deployment Status by Policy Enforcer

On each tab, when you select an enforcer in the Host list, the Contents list dis­plays all policies and components deployed on it.

When you select a policy or component in the Contents list, the item’s definition appears in the Details pane.

Version History

As another alternative, you can check which policy enforcers have received deployment of a particular policy or component, when the deployment occurred, and whether more than one version of a policy has been deployed. This can happen when you make some changes to a policy and then deploy it to the network, but the new version cannot be deployed to one or more enforcers—for example, if the host is a laptop that is temporarily disconnected from the network.

To view this information, select a policy or component, then select Actions > Version History. If the command is disabled, it means you have selected an object that has never been deployed. This command opens the Version History window, as shown in the following figure.

 

Version History window

The left pane of this window displays a list of currently scheduled and past deployments of the selected policy or component. Each item in this list repre­sents a version of the selected policy or component. You can select any item in this list and view details, in the information pane on the right. That pane con­tains two tabs:

·        The Definition tab displays the detailed definition of the selected ver­sion.

·        The Deployed Policy Enforcers tab displays a list of hosts where the selected version has been deployed, and the type of each host: desktop, file server, or portal. Here too, file servers will include both Windows and Linux, without distinction. Also, both Windows Desktop Enforcers and Out­look Enforcers are considered desktop type.

Policy Audit Logging

Policy Studio supports the ability to audit which users made changes to policies in each stage of the policy life cycle, and when the changes were submitted. You can view the following information in four locations in Policy Studio:

•Policy creation date, time, and user ID

•Policy modification date, time, and user ID

•Policy submission date, time, and user ID

•Policy deployment date, time, and user ID
