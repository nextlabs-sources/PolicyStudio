Working with Components

The following table provides high-level instructions for how to work with compo­nents in Policy Studio.

How do I . . .

Details

Create a new component?

PA: In the component bin, expand one of the component lists, click the New button, provide a name, and define the component in the component editor pane.

Create a component that represents USB devices?

PA: To cover all removable devices: Use the pre-defined Removable Media component, in the Documents component bin.

To cover specific types of devices: Create a new Document component and, in the Document Path field, define it with the special "device://" syntax. For details, refer to the Policy Design Guide.

Delete an existing component?

PA: Right-click on the component and select Deactivate (if it is active), then right-click and select Delete. This command is also available in the Edit menu. You can select multiple components, using Ctrl + click, for deletion. To delete multiple components, the selected components must either be in Inactive or Draft (This object has not been deployed) status.

View more than one component at a time?

PA: Open two components, click to grab the tab of one, and drag it to the bottom or to either side of the editing pane. You can tile multiple components this way.

Rename a component?

PA: Right-click on the component and select Properties. On the General tab, type the new name, then click Save. See Renaming an Object.

Deploy a new or changed component?

PA: Individual components: click the Submit button if not yet submitted; then the Deploy button. Or, deploy all submitted components using the Deploy or Deploy All commands in the Actions menu.

Manually control where I deploy a component?

PA: Right-click on the component and select Set Deployment Targets. This command is also available in the Actions menu.

Find out which policies a component is used in?

PA: Right-click on the component and select Show Policy Usage. This command is also available in the Actions menu.

Find out which enforcers a component is deployed to?

PA: Select the component, then open the Deployment Status window (under the Tools menu). This window has four tabs, listing the File Server Enforcers, Desktop Enforcers, Activity Directory Enforcers, or Portal Enforcers where the component is deployed.

Control who can use a component in their policies?

PA: Right-click on the component and select Properties. On the Access Control tab, add or remove users or user groups from the list. For each user or group, you can set individual permissions. See Changing Access Rights.

Control how others can use a component in their policies?

PA: Right-click on the component and select Properties. On the Access Control tab, choose the user or group you want to edit, then select or deselect any of the six available Permissions. See Changing Access Rights.

Find out what version of a component is deployed?

PA: Right-click on the component and select Show Deployed Version. This command is also available in the Actions menu. See Comparing to the Deployed Version.

View previously deployed versions of a component?

PA: Select the component, then select Actions > Version History. (Disabled if only one version of the component was ever deployed.)

Know whether any applications are being globally ignored?

PA: Right-click on the special Monitor All Applications—Exceptions component, and select Show Deployed Version. Any applications displayed in this list will be ignored by all currently deployed policies. If this component is not deployed, no applications are being ignored.

Test whether I have defined a component correctly?

PA: Select the component, open the Preview window (WIndow > Preview), then click the Go button. Not available for action components.

Enroll new entities to use as components?

These procedures must be performed by a system administrator, using a set of special utilities. Details are provided in the Control Center Administrator’s Guide.

Enroll SharePoint sites?

SharePoint sites are not enrolled, only SharePoint groups. This is done outside of Policy Studio; refer to the Control Center Administrator’s Guide.

Define a component to cover a directory in MyDocuments?

PA: Type the keyword [MyDocuments] in the membership field of a document-type component. See Special Keywords.

Exclude an application from all policy enforcement?

PA: Use the special application component, Monitor Specific Applications (see Application Monitoring).

Monitor activity on only specified applications, ignoring all others?

PA: Use the special application component, Monitor All Applications—Exceptions (see Application Monitoring).
