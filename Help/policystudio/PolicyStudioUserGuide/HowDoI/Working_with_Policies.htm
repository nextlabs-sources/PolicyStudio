Working with Policies

The following table provides high-level instructions for how to work with poli­cies in Policy Studio.

How do I . . .

Details

Create a new policy?

PA: Select a folder in the policy tree, and click the New Policy button. Provide a name, and define the policy in the editor pane.

Delete an existing policy?

PA: Right-click on the policy in the policy tree and select Deactivate (if it is active), then right-click and select Delete. Both commands are also available in the Actions menu. You can select multiple policies, using Ctrl + click, for deletion. To delete multiple policies, the selected policies must either be in Inactive or Draft (This object has not been deployed) status.

View more than one component at a time?

PA: Open a two components, click to grab the tab of one, and drag it to the bottom or to either side of the editing pane. You can tile multiple components this way.

Rename a policy?

PA: Right-click on the policy and select Properties. On the General tab, type the new name, then click Save.

Manually designate where I want to deploy a policy?

PA: Right-click on the policy and select Set Deployment Targets. This command is also available in the Actions menu.

Deploy a new or changed policy?

PA: For individual policies, click the Submit button if not yet submitted; then the Deploy button. Or, deploy all submitted policies with the Deploy All command in the Actions menu.

Cancel a submitted policy?

PA: From the Tools menu, select Deployment History. In the Deployment History window, select the deployment containing the policy you want to delete, and click the Delete Selected Policy button. See Canceling Deployment.

Find out which enforcers a policy is deployed to?

PA: Select the policy, then open the Deployment Status window (Tools --> Deployment Status).

Control who can use a policy?

PA: Right-click on the policy and select Properties. On the Access Control tab, add or remove users or user groups from the list. For each user or group, you can set individual permissions. See Changing Access Rights.

Control how others can use a policy?

PA: Right-click on the policy and select Properties. On the Access Control tab, choose the user or group you want to edit, then select or deselect any of the six available Permissions. See Changing Access Rights.

Find out what version of a policy is deployed?

PA: Right-click on the policy and select Show Deployed Version. This command is also available in the Actions menu.

View previously deployed versions of a policy?

PA: Select the policy, then Actions menu, Version History command. (Disabled if only one version of the policy was ever deployed.)

Construct a policy covering all removable media?

PA: Use the predefined document component RemovableMedia, for either the source or target document of your policy. See Removable Media.

Write a policy that controls access and use of file directories, rather than files?

PA: Create a Document component based on the special Include Only Directories property (see About the Include Only Directories Property).

Export policies I have already defined from one E-DLP implementation, to another?

PM: Select a folder in the object tree, or the individual policies in the object grid, click the respective Export icon, and specify a path and name for the output XML file. See Exporting Objects.

Import policies into my E-DLP implementation?

PM: Select a target folder in the object tree, click the Import command in the File menu, and specify the XML source file you want to import from. See Importing Objects.

Export components I have already defined from one E-DLP implementation, to another?

Components cannot be exported by themselves, but only in combination with parent policies.

Generate a definition report describing the definitions of policies?

PM: Select one or more policies in the object grid, and click the Generate PDF icon. See Removable Media.

View a list of all policies in a certain state?

PM: Select one of the five state tabs.
