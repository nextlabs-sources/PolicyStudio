Cloning Policies

For various reasons, you may need to create one or more policies that are very similar to existing ones, but with only a minor change. In such cases, it may be more convenient to make a copy of the existing policies and then make the desired changes in the copies, rather than build the new policy from scratch. This is referred to as policy cloning. When you clone a policy, any subpolicies within it are also cloned.

To clone a policy,

  1. Open the appropriate folder in the policy tree and select the policy you wish to clone. If you want to clone all the policies in a folder, select the folder and clone that.

  2. Either right-click and select Duplicate, or select Edit > Duplicate. You can also use the Ctrl + D shortcut.

  3. In the Target field, select the location where you want the duplicate pol­icy or folder to be saved. You can save a policy in a folder (as a top-level policy) or in another policy (as a subpolicy). If saving a policy within another policy, both policies must be of the same type; for example, a document policy can be saved only within another document policy.

  4. If you are cloning a policy, you can assign a new name in the New Name field. A new name is required if you are saving the new policy in the same folder as the original policy. If saving to a different folder, you can use the same policy name.

  5. Click the Save button to clone the selected policy or folder.

Note: The components used by cloned policies are not themselves cloned; rather, the cloned copy and the source policy share the original com­ponents. If this does not suit your needs—that is, if you want to change the definition of one or more components as well as the policy—you need to clone the components as well. For details, see Cloning Compo­nents.
