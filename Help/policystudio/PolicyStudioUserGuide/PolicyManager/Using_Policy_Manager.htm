Using Policy Manager

Once you have created components and policies in Policy Author, you can switch to the Policy Manager window to monitor them in various ways. To toggle to Pol­icy Manager, select Policy Manager in the Window menu or use the Ctrl + Tab shortcut.

The State Tabs

Policy Manager is organized into five main tabs: Draft, Pending, Deployed, Deactivated, and All. We will refer to these as the state tabs. They work as fil­ters for the Object Grid pane, which displays only components and policies that are in the state represented by the currently selected tab. Unless the All tab is selected, the Policy Manager does not display a list of all currently defined poli­cies and components in a given folder, but only those in the selected state. To view all currently defined objects in all states, you can select the All tab. (You can also toggle to the Policy Author object tree, to view all defined objects in Edit mode.)

 

State tabs in Policy Manager

Structure

Whichever State tab is selected, the Policy Manager window displays three panes:

·        The Object Tree pane (A)

·        The Object Grid pane (B)

·        The Details pane (C)

These panes are illustrated in the following figure.

 

The Policy Manager window

It is helpful to think of all your work in Policy Manager as a combination of filter­ing and drilling down. You can filter along two axes: by the State tabs, which represent hard-wired categories, and by policy folders, which are customizable and reflect whatever structure you defined for organizing your policies. Once you have used one or both of these filters to focus on a set of objects, you drill down to select individual objects in the Object Grid pane, and then view the definitions of those objects in the Details pane.

The following sections describe each of these panes in more detail.

Using the Object Tree

The Object Tree pane, at the left of the window, is used only for navigating through the folder structure you have created to store your policies (when you are working with policies), or for selecting specific component types (when you are focusing on components). As the following figure shows, the Object Tree pane displays the current hierarchy of folders, and a list of component types, represented by different icons. The policies and components themselves are dis­played in the object grid, not in the tree.

 

Object tree in Policy Manager

You use the object tree as a navigation tool, so you can view and manage objects in the other two panes. When you select a folder in the tree, the poli­cies it contains are displayed in the object grid, where you can expand them to view their child objects. If you select a component type icon in the tree, the object grid displays the components of that type. That is, the object tree is for selecting only groups of objects; it can expand to show subfolders, but not indi­vidual policies or components.

As mentioned earlier, the first four state tabs serve as filters that allow you to view only objects of a given state: Draft, Pending, Deployed, or Deactivated. If you select one of these tabs and then select a folder in the Object Tree, all pol­icies and components in that folder and also in that state display in the list in the Object Grid pane. For example, suppose you have a folder called HR con­taining your HR data policies. If you select the Deployed state tab then the HR folder in the object tree, the object grid displays all HR policies currently in Deployed state. Draft, Pending, and Inactive HR policies, if any, are not dis­played.

Components are a bit different. If you select one or more of the component type icons in the tree, the object grid will display all components in that state, regardless of the policies where they are used—that is, both in HR policies and all others as well.

The fifth tab, All, allows you to view policy objects in all states, for the selected policy folder, or the selected component type.

Management Functions

The icons displayed above the object tree control various management func­tions.

 

Object Tree function icons

There are six function icons:

·        Export as ACPL: Launches the export window, which allows users to export already defined policy objects in ACPL (as an xml file). See Export­ing Objects.

·        Export as XACML: Launches the export window, which allows users to export policy objects as XACML (a xacml file). See Exporting Objects.

·        Refresh: Refreshes the display of all objects in the object tree, to reflect any changes made by other users.

·        Properties: Opens the Properties panel for the selected folder, where you can reset user permissions. Enabled only when a folder is selected.

·        Create Folder: Creates a new folder, as a child of a selected folder. Enabled only when a folder is selected.

·        Delete Folder: Deletes the selected folder. Enabled only when a folder is selected, and the folder must be empty.

If any of these icons appears as a pale, washed-out image, it means it is disabled due to the state of the selected folder in the tree. For example, because only empty folders can be deleted, when you select any folder that is not empty, the disabled version of the Delete Folder icon appears.

More details on these functions are provided below. Note that all these functions except Import are also available when you right-click on any folder in the tree.

Setting Properties

When you right-click a folder and select Properties, the Properties window for that folder displays. This has two tabs, General and Access Control. On the Access Control tab, you can define an access control list (ACL) that governs who can access its contents. This same tab is also available in Policy Author; for details, see Viewing Properties.

On the General tab, all properties are read-only if the folder contains any items. If a folder is empty, the Name field here will be editable, and you can assign a new name if you wish.

Creating Folders

In addition to navigation, you can also use this pane to create and organize pol­icy folders. You can expand or collapse folders to display any subfolders, and you can also create and delete folders and subfolders. These are important organizing tools because all policies must be created and saved in some folder, and because they are essential for managing large numbers of policies. (The list of component types is not expandable or editable.)

Whenever you create folders and set permissions for them here, all changes are reflected in Policy Author. In practice, a Policy Administrator might create the overall (empty) folder structure here and assign appropriate permissions to mul­tiple Policy Analysts, who would then use Policy Author to define the policies in their assigned areas—one analyst to define Financial Research document poli­cies, a different one for Merger & Acquisition document policies, and so on.

To create a new folder in Policy Manager:

  1. Select the root where you want to create the new folder. This can be either the top folder, labeled Policies, or any subfolder in the tree.

  2. Click the New Folder icon at the right just above the tree; or right-click and select New Folder.

  3. Type a name for the folder.

Note that you can also delete folders in Policy Manager, but only if they are empty. To delete a folder, do one of the following:

·        Select it and click the Delete Folder icon, at the right above the tree.

·        Right-click on the folder and select Delete Folder.

If these commands are disabled, it means the selected folder cannot be deleted because it contains one or more policies.

Again, any changes made in Policy Manager are reflected in Policy Author.

Using the Object Grid

The Object Grid, in the upper right quadrant of the Policy Manager window, dis­plays the contents of whatever folder or folders are selected in the Object Tree—filtered by whichever State tab is active. If you choose the root folder, it lists all policies in the tree, in the given state; if you select a sub-folder, it shows all such policies in that folder and its sub-folders. Note that the folder structure is not represented in the object grid, regardless of how many levels of subfold­ers there are; all policies in nested folders are flattened to one list.

 

Object grid pane in Policy Manager

Because the Object Grid may display policies stored in more than one folder, when you select a policy in the list, the folder to which it belongs is underlined in the object tree. This feature makes it easy to track where each policy is stored, as you work with it in the object list.

For each policy in the list, all its components are also displayed as child objects. By default they are initially collapsed under the policy, but you can expand them. For clarity, each component type is indicated by a distinct icon.

Of course, if the All tab is selected, all contents of the selected folder will be listed in the Object Grid, regardless of what state the objects are in.

You use this grid to select individual objects, in order to view their definition details in the Object Details pane, below the grid. You can use CTRL + click or SHIFT + click to select more than one object at a time, but the details pane can only show details for one at time. If you select more than one object in the grid, the details reflects the last one selected.

The Search Tool

Whichever State tab you have selected, a search tool displays above the Object Grid. This can be used to search for specific objects, by name. To run a search, type a string of one or more characters into the field, then click the Search but­ton. The grid will then display any objects that start with the specified charac­ter string. When searching, note the following points:

·        The search returns only objects that are in the currently selected state (tab).

·        The search returns only the type of object selected in the Object tree. For example, if you have selected a policy folder, the resulting list will contain only policies in that folder; or if you select the User components icon, it will contain only user components.

·        The search tool does not support wild cards—only the exact string you pro­vide.

Function Icons

As shown in the previous figure, several function icons display at the top right corner of the object grid:

·        Export as ACPL: Exports the selected objects in ACPL format to an xml file that can be imported into another instance of Policy Studio, using the Import command. For details, see Exporting Objects.This function is also available by right-clicking on one or more objects in the list.

·        Export as XACML: Exports the selected objects in XACML format to a XACML file for import into other Policy Management Systems. For details, see Exporting Objects.This function is also available by right-clicking on one or more objects in the list.

·        Generate PDF: Creates a Definition Report of all objects selected in the grid, as an easy-to-read PDF file. This feature is very convenient for audi­tors or other analysts who need a quick, clear look at how policies and components are defined. You can save multiple objects to the same Defi­nition Report, and you can generate reports of policies, components, or both together.

·        Change State: Changes selected objects from their current state to the next logically appropriate state. Whether this icon displays, and how it works, depends on which state tab is selected:

o       Draft tab = Does not display

o       Pending tab = Changes selected objects from Pending Deployment to Deployed

o       Deployed tab = Changes selected objects from Deployed to Submit­ted for Deactivation

o       Deactivated tab = Does not display

o       All tab = Does not display

Note: For a general discussion of object states, see The Object State Model.

Grid Information

For each object displayed in this grid, the following information is provided:

·        Version: Displays the version of the object. For more information, see Using Versions.

·        Submitted Time: Date and time when the current version was submitted, either for deployment or for deactivation.

·        Submitted By: User who submitted the current version for either deploy­ment or for deactivation.

·        Status: Shows current status of the selected object. This actually refers to the status of the specific version selected, since more than one version of the same object may be listed in this grid. For more information, see About Object Status.

Right-Click Commands

When you select one or more objects in the grid, you can right-click to perform several operations on them. These include:

·        Export as ACPL: Exports the selected objects in ACPL format to an xml file that can be imported into another instance of Policy Studio, using the Import command. For more information, see Exporting Objects.

·        Export as XACML: Exports the selected objects in XACML format to a xacml file for import into other Policy Management Systems. For more information, see Exporting Objects.

·        Version History: Opens the Version History window, with version history information for the selected object. This is the same window as available in Policy Author. For more information, see Version History. Disabled when more than one object is selected. Also, if a version window does not open, it means the selected object has no versions other than the present one.

·        Open: Toggles to Policy Author, with the selected object open in the object editor. Disabled when more than one object is selected.

·        Duplicate: Make a copy of the selected object and save the copy in a specified location. For more information, see Cloning Policies.

Using the Details Pane

Just as the Objects Grid expands on whatever folder is selected in the Objects tree, the Details pane drills down to show various details and properties of whatever object is selected in the Objects Grid. When you select either a policy or a component in the objects grid, the details about it are displayed in the Details Pane. This pane contains the Definition tab and, depending on the type of object selected, possibly a Preview tab and a Hosts tab.

The Definition Tab

The Definition tab, on top by default, displays a plain-English description of the definition of whatever policy object is selected in the Objects Grid.

 

Details Pane, Definition Tab

For policies, it shows the structural definition of the policy, as defined in Policy Author’s editing pane: Enforcement, Subject, Action, Resource, Context, Sub­policy, and Obligation. For components, it shows whatever definitional attri­butes and properties are appropriate, given the component type.

The Preview Tab

This tab provides a list of the actual entities that would be represented by the component selected in the grid pane. When you select a component in the objects grid, this tab displays all the actual entities in your system that would be represented by that component. For instance, for a User component defined as All Users In Property LastName = W*, this tab would show a list of all enrolled users whose last names begin with W. (This tab’s function is identical to that of the Preview pane in Policy Author. For more information, see The Preview Pane).

The Hosts Tab

This tab displays a list of all enforcer hosts where the policy object selected in the Grid pane is deployed. It provides the host name, the user who deployed it to that host, and the type of enforcer running on the host—Desktop or Server.

 

Details pane, Hosts tab

Note that this tab only displays when the Deployed state tab is selected, since it does not make any sense for objects in the other three states—Draft, Pending, or Deactivated.

Menu Commands

In Policy Manager, the same menus are available as in Policy Author. Some of the commands may be disabled (greyed-out) at any given time, depending on whether they are logically valid for the type and state of the currently selected policy object. For example, if no object is selected, all but one of the Actions menu items are disabled; or if the selected policy is not submitted for deploy­ment, the Deploy command is disabled.

Actions

Modify

Changes the status of the selected component or policy to Draft. Disabled if no object is selected, or if the selected object is not in a state to be modified.

Submit

Submits the selected components or policies for changing from one status to another—for example, from Draft status to Submitted for Deployment. Disabled if no object is selected, or if any of the selected objects is not currently in Modify state.

Deploy

Deploys the selected component or policy. Disabled if no object is selected, or if the selected object has not been submitted for deployment.

Deploy All

Deploys all currently submitted components or policies.

Deactivate

Changes the status of the currently selected policies or components from Active to Deactivated. Disabled if no object is selected, or if any of the selected objects is not currently in Active state.

Disabled.

Show Deployed Version

For the selected component or policy, displays the detailed definition of the version that is currently deployed. Disabled if no object is selected, or if the selected object is not currently deployed.

Check Dependencies

For the currently displayed component or policy, runs a value check and displays all the components that are required by (that is, referenced inside) the component or policy.

Set Deployment Target

Allows you to manually specify which enforcement point the selected policy or component will be deployed to. You can use this feature as an alternative to Control Center’s Auto-deployment capability.

Version History

Opens the Version History window for the currently displayed component or policy. This window displays a list of all previously deployed versions, starting with the most recent one. Versions are identified by date and time stamp. This command is disabled if the selected object has never been deployed.

Update Computers with Enforcers

Updates the definition of the special Computer component, All Computers with Enforcers, to include any newly-installed enforcers in the network. For more on this component, see Computer Components: Membership.
