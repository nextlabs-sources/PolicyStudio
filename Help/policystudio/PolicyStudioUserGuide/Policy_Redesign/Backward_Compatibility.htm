Backward Compatibility

All policies, except Dynamic Access Control (DAC) policies, created in earlier versions of Policy Studio will continue to run without any changes. You do not have to modify or redeploy any of your document, communications, or device control policies.

Policy Studio 7.0 and later do not support the following tasks:

·        Changing the enforcement type of existing Allow only policies.

·        Creating new Allow only policies.

·        Creating new Monitor policies.

·        Modifying or deploying existing DAC policies.

Deny Policies

In Policy Studio 7.0 and later, you view and edit existing Deny policies in the same way as in previous versions. The behavior of Deny policies is the same in the current and previous releases.

Allow only Policies

In Policy Studio 7.0 and later, you can view and edit existing Allow only policies. You can edit every item in the policy, except for the enforcement type. In other words, you cannot change an Allow only policy into a Deny or Allow policy.

Monitor Policies   

In Policy Studio 7.0 and later, all existing Monitor policies are automatically con­verted to Allow policies. In version 7.0 and later, Monitor is not necessary because Allow does the same thing, and also supports the definition of condi­tions. You can modify your Monitor, now Allow, policies to include conditions.

DAC policies

You must recreate all your DAC policies.
