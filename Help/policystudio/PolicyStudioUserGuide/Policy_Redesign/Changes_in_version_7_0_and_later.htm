Changes in version 7.0 and later

Version 7.0 changes the way policies are designed. If you used earlier versions of Policy Studio, you see the following changes when you build policies in Policy Author:

·        Different enforcement types

Version 6.x

o       Deny: If the event described in the policy occurs, do not permit the listed Subjects to perform the task specified in the policy, but allow all others to do so.

o       Allow only: If the event described in the policy occurs, permit the listed Subjects to perform the specified action, and prevent all oth­ers from doing so.

o       Monitor: Always permit the specified action, and perform the spec­ified obligations whenever the action occurs.

Version 7.0 and later

o       Deny: This behavior is the same in all versions.

o       Allow: If the event described in the policy occurs, permit the listed Subjects to perform the action specified in the policy. Allowing a set of Subjects to perform a task does not mean that others are blocked from performing that action.

·        Different way of structuring or composing logic in policies. In version 7.0 and later, policies can be nested within policies, where each subpolicy typically expresses an exception to its parent policy. In earlier versions, you could not create subpolicies; exceptions were defined within a policy in more complex ways, for example, combining multiple subjects using multiple negative clauses, or defining multiple conditions. For more infor­mation about subpolicies, see Creating Subpolicies.
