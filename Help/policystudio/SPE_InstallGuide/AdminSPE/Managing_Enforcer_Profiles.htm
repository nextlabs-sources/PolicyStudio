Managing Enforcer Profiles

All policy enforcers are governed by a number of configuration settings that control such aspects as logging behavior, heartbeat rates, tamper-prevention passwords, and network configuration. These are assigned default values when you first install an enforcer, but they can be changed manually at any time. To simplify this, Administrator allows you to create named sets of configuration settings, which you can then assign to one or more enforcers in your network. These are referred to as enforcer profiles, and you manage them in Administrator, with the settings on the Policy Enforcer Configuration tab.

 

The settings controlled by enforcer profiles include the following:

1       Which ICENet Server the enforcer will use to communicate with the Pol­icy Server

2       How often the enforcer sends heartbeat signals to the Policy Server to indicate it is operating normally

For more details on defining and using enforcer profiles, refer to the “Introducing Administrator” chapter in the Control Center Administrator’s Guide.
