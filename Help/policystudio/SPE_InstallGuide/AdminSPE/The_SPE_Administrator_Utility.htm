The SPE Administrator Utility

The Entitlement Manager for SharePoint includes a command line utility called the SPE Administrator, that allows administrators to set certain configuration settings that control the enforcer’s content trimming features. The following tables describe the commands available for this utility and the available arguments for each command. Note that these tables are just summaries; for each command, more detailed use instructions are provided elsewhere in this manual.

Command

Purpose

Command String Syntax

securitytrimming

Enables or disables security trimming feature

ce_spadmin -o securitytrimming {{-enable | -disable | -remove} [-url <SiteCollectionUrl>] | {-setcachetime <integer> | -clearcache }}

searchresulttrimming

Activates and deactivates search result trimming

ce_spadmin -o searchresulttrimming {-install | -uninstall} -id <Integer> [ -rulepath <crawl rule URL>] [-ssp <Shared Service Provider Name>]

Argument

Used With

Description

-o

All

Used with all commands to specify the command name. Valid values are the three commands describe in Table 5-2, above.

-enable

All

Enables the feature specified by the command, for the specified site collection. Valid for all SPE Administrator commands.

-disable

All

Disables the feature specified by the command, for the specified site collection. Valid for all SPE Administrator commands.

-remove

searchresulttrimming

Removes the security trimming feature from the SharePoint server altogether. This is required whenever you uninstall the Entitlement Manager for SharePoint.

-force

securitytrimming

Runs the specified command in Force Check-in mode, which automatically checks in any master pages or layout pages for the specified site, that are currently checked out, so it can apply the configuration changes to them.

-url <URL>

All

Specifies the URL of the site collection to which you want to apply the specified command.

-setcachetime <integer>

securitytrimming

The duration, in minutes, that the search trimming cache will retain search result data. Search data is cached in order to improve performance of subsequent searches; all data that is older than the amount specified here is purged.

-clearcache

searchresulttrimming

Purges all data from the search trimming, regardless of its age.

-install

searchresulttrimming

Enables the search result trimming feature, once required configuration has been performed.

-uninstall

searchresulttrimming

Disables the search result trimming feature.

-ssp

searchresulttrimming

Name of the shared service provider—for example, SharedServices1—to which you want the search result trimming feature to apply.

-id <integer>

searchresulttrimming

Unique identifier for each security trimmer instance. Each trimmer corresponds to one crawl rule path. Valid values = 0 to 2147483647; default = 100

-rulepath

Search Result only

The crawl rule path to which this trimmer maps. Each rule path requires its own trimmer instance, with a unique ID.
