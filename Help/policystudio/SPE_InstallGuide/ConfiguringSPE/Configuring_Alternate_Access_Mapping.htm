Configuring Alternate Access Mapping

Another necessary configuration step is creating Alternate Access Mappings, which ensures that a fully qualified domain name always defaults for other URLs users might enter. You do this in SharePoint by mapping a fully qualified name as the “Internal URL” for each web zone.

The procedures for this configuration are slightly different In SharePoint 2007and In SharePoint 2010. They are discussed in separate sections below.

In SharePoint 2007

  1. Access the SharePoint server at Start > All Programs > Microsoft Office Server > SharePoint 3.0 Central Administration.

  2. In the Global Configuration area of the Operations tab, click Alternate Access Mappings.

    3.  

    Alternate Access Mappings

  3. In the Alternate Access Mappings window, click Add Internal URLs.

    4.  

    Alternate Access Mappings window

  4. In the URL, protocol, host and port field, enter the fully qualified domain name of the site. In the example in the following figure, this is “http://lab01-sps15.lab01.cn.nextlabs.com.”

  5. Select the Zone for the URL.

    6.  

    Adding an URL

  6. Click Save to save the mapping.

  7. Repeat these steps to enter a fully qualified domain name as the "Internal URL" for every configured web zone on your service. For example, you would need to do this for the “internal” zone and the “external” zone.

In SharePoint 2010

  1. Access the SharePoint server at Start > All Programs > Microsoft SharePoint 2010 Products > SharePoint 2010 Central Administration.

  2. In the System Settings area, click Configure alternate access mappings.

    3.  

    Alternate Access Mappings

  3. In the Alternate Access Mappings window, click Add Internal URLs.

    4.  

    Alternate Access Mappings window

  4. In the Alternate Access Mapping Collection, select SharePoint-80.

  5. In the URL, protocol, host and port field, enter the fully qualified domain name of the site. In our example in the following figure, this is “http://lab01-sps15.lab01.nextlabs.com.”

  6. If applicable, select the Zone for the URL.

    7.  

    Adding an Internal URL

  7. Click Save to save the mapping.

  8. Repeat these steps to enter a fully qualified domain name as the "Internal URL" for every configured web zone on your service. For example, you would need to do this for an “internal” zone and an “external” zone.
