Configuring Special Actions

The Entitlement Manager for SharePoint has two special actions, Upload and Download, that you must manually configure before you can use them in policies. These actions will not be available in Policy Studio until you add them.

Note: For more information on which actions should be associated with dif­ferent kinds of SharePoint policies, see What Actions Can Be Enforced?

Follow these steps:

  1. Use Notepad to open the file called SPE_Actions.xml, which is supplied in the Support directory of your Compliant Enterprise installation CD.

  2. Open the main configuration file, configuration.xml, on the Control Cen­ter host. By default it is located at <installDirector>\server\config­uration.

  3. Paste the two special actions from the SPE_Actions.xml into the main con­figuration file. They must be placed inside the <ActionList> section, which is within the <Obligations> section, as shown in the figure below.

  4. Save your changes in the file and restart the Control Center Service.

  5. After the system restarts, test the configuration by opening Policy Studio and creating a New action component. The two new actions should appear in the list of basic actions in the left side of the chooser.

To configure these obligations, perform the following steps:

  1. Open your Control Center configuration file, configuration.xml.

  2. Open the obligation source file and locate the four download-related sec­tions.

  3. Copy these four sections from the source file to the <Obligations> section of your configuration file, and save the changes.

  4. Restart your Policy Server (and Policy Studio, if it is running).

  5. Test the registration by creating a new policy in Policy Studio and clicking the Custom Obligations checkbox. The four new obligations should display in the list of available obligations.

The XML code snippets for these obligations are available in a source file called product.xml, which can be obtained from NextLabs support. Two examples of these snippets (for tagging obligations) are shown in the figure below.

Configuring   Settings Page Components

The Entitlement Manager for SharePoint lets you write policies that control how users can use the setting pages and other administrative controls within SharePoint itself. This enables you to set very fine-grained policy-based permissions not only for who can access which locations on SharePoint, but also for how users with administrative permissions can and cannot use SharePoint’s administrative settings to allow access to other users.

Of the ten pre-built settings page components we described above, only one—SharePoint Setting - Users and Permissions—is already available by default in the Portal component bin when you install Policy Studio. The others must be imported manually from an XML source file.

Note: For an example of how to incorporate these components into policy, see Example Policy:   Permission Change Auditing .

Follow these steps:

  1. Copy the source file, SharePoint_Components.xml, from the Support folder of the Compliant Enterprise installation CD to any location visible to your Policy Studio host.

  2. Open Policy Studio and toggle to the Policy Manager interface.

  3. In the File menu, click the Import command.

  4. Browse to the XML source file, and click the Import button.

Configuring SharePoint Site Columns for  Tagging Policies

Before you are ready to construct content aware policies for automatic tagging of SharePoint documents, you must perform two configuration steps in SharePoint: designate or create a site or library that is the target site for the policy, and add a column to the site named for the tag that you are going to create. You should not change the default settings that apply for new columns: single line of text, "No" selected for require the column has information and for the enforcement of unique values, etc.
