Installing the Entitlement Manager  

Installing the Entitlement Manager for SharePoint involves two steps:

1       Installing the Policy Controller

2       Installing the Policy Adapter

Installing the Policy Controller

To install the Policy Controller directly on the local host, insert the installation CD into the D: drive to launch the installation CD splash screen. (If the splash does not launch automatically, double-click on the autorun.exe file on the CD root.) Click the Install Policy Controller link to launch the install wizard.

  1. On the Licence Agreement screen, read the license agreement, select the “I Accept . . . ” radio button, and click Next.

    2.  

    License Approval Screen

  2. On the Destination Folder screen, specify the path and directory where you want to install the Entitlement Manager, and click Next.

    3.  

    Specifying a Destination Folder

  3. On the ICENet Server Location screen, use the drop-down list to select the ICENet Server the Policy Controller will use for its connection when it first registers itself with the Policy Server. (After it registers, it will use whatever ICENet Server is specified in the default profile.) This list dis­plays all the ICENet Servers detected in the network; however, you can also type in a different host name (if your desired ICENet Server is not cur­rently running).

    4. If you are using load-balanced ICENet Servers, type in the virtual IP address or virtual host name of the load balancer here.

    Note: In order to view all active servers in this list, you should make sure the autodiscovery port (19888) on the server host is not blocked by a fire­wall. For details, refer to the Control Center Installation Guide.

     

    Specifying an ICENet Server

  4. If this is your first installation of the Policy Controller on the host, a dialog box may display warning you that you are allowing an unknown publisher to make changes to your computer. Allow the changes to be made, and if the option exists, click Always Trust for the publisher.

  5. In the final Wizard screen, click Install. A notification message lets you know when installation is complete.

Installing the Policy Adapter

To install the SharePoint Policy Adapter on a server locally, perform the following procedure.

Note: Be sure you the user permission and other pre-requisites before install­ing the Policy Adaptor. See Before You Start.

  1.  Locate the installer file for the SharePoint Server and copy it to the tar­get host, or to a network shared folder.

  2. For SharePoint Server 2007, launch the install wizard by double-clicking the installer file. For SharePoint Server 2010, run a command prompt as Administrator, and run the installer file from within the command prompt.

  3. When the wizard’s splash screen appears, click Next.

  4. On the Licence Agreement screen, read the license agreement, choose “I Accept,” and click Next.

  5. On the Destination Folder screen, accept the default install directory, or click the Change button to specify a different location, then click Next.

  6. On the ICENet Server Location screen, use the drop-down list to select the ICENet Server this Entitlement Manager will use for its connection when it registers itself with the Policy Server. (After it registers, it will use whatever ICENet Server is specified in the default profile.) This list will display all ICENet Servers detected in the network; however, you can also type in a different value (if your ICENet Server is not currently running).

  7. If you are using load-balanced ICENet servers, type in the virtual IP address or virtual host name of the load balancer here.

  8. On the next screen, click the Install button to begin the installation pro­cess.

  9. After the installation finishes, the IIS running on the install host will have to be restarted. On the next screen, click Yes to restart it now, or No if your prefer to restart it at a later time. (Note that this refers specifically to restarting the IIS; there is no need to reboot the host machine.) You will not be able to use the Entitlement Manager until you do this.

Is the Entitlement Manager Running?

There are two ways to determine if the Entitlement Manager has been installed properly and is running: locally on the host where it is installed, and from the Control Center Administrator.

Checking Locally

To determine locally whether or not Entitlement Manager for SharePoint is running on a given host, simply check whether the Control Center Enforcer Service is running as a Windows Service. Once installed it will display in either Started or Stopped state.

Checking in Control Center Administrator

To confirm remotely whether Entitlement Manager for SharePoint is running, perform the following steps:

  1. Open the Control Center Administrator web application in your web browser.

  2. Go to the Status tab, and click the Policy Enforcer Status link.

  3. If you want to view all installed desktop enforcers, select the All Server Enforcers filter in the Show field (A in the figure below).

  4. If you want to check a specific host or host group, type the name in the Search By Host field (B in the figure below).

  5. Be sure the Enforcers with Warnings Only checkbox is not checked.
