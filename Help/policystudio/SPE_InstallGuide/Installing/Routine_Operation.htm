Routine Operation

Once it has been installed, the Entitlement Manager for SharePoint runs continuously, and should require little or no attention from an administrator. As we described above, it consists of two logical parts: the Policy Controller, which serves as the policy decision point (PDP), and the Policy Adapter, which serves as the policy enforcement point (PEP).

Processes

The Policy Controller runs as a standard Windows service, and will be visible as an item in the list in the Services management window, called Control Center Enforcer Service. If this service ever stops for an abnormal reason, the Windows operating system will automatically restart it immediately, and the events will be logged.

The Policy Adapter is not an active process but rather a set of DLLs that are written to the required locations on the server, and are then called by the IIS whenever requests come in for access to SharePoint content. The Policy Adapter makes a decision on whether the request is something that might be covered by a current policy, and if so it passes the information to the Policy Controller, where rules for enforcing currently deployed polices at that server. The Policy Controller passes its enforcement decision—Allow or Deny—back to the Policy Adapter, which passes it to the IIS.

Uninstalling or Changing Entitlement Managers

Entitlement Managers for SharePoint use the standard Repair/Remove procedure common to all Compliant Enterprise Entitlement Managers. For details, see Uninstalling, Repairing, or Modifying. Note the following important points about uninstallation:

The users who uninstalls the Entitlement Manager should fulfill all the pre-requisites as the user who installs it. (See Installer User Permissions.)

You must uninstall both the Entitlement Manager for SharePoint and the Policy Controller; they are listed separately in the Add/Remove Programs list.

Just as with installation, the installer .MSI file used for upgrade must be launched in a command prompt that is run as Administrator.

If you have activated the search result trimming feature (see Search Result Trimming), you must disable it before you uninstall either the Entitlement Manager or the Policy Controller. To do this, use the Entitlement Manager for SharePoint Administrator utility, with the command line:

ce_spadmin -o searchresulttrimming -uninstall

For details on this using this utility, see The SPE Administrator Utility.
