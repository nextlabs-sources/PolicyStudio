About SharePoint Policies

For policy design purposes, you can think of the Entitlement Manager for SharePoint as capable of enforcing four kinds of policies:

1       Data Classification Policies

2       Access and Usage Control Policies

3       Data Protection Policies

4       Auditing Policies

Each of these policies types, along with example use cases, is described in more detail below.

Note: Example policies in each of these categories are presented in Using the Entitlement Manager for SharePoint.

 

Data Classification Policies

The first step of Entitlement Management is to apply Classifications to data. Entitlement Manager for Sharepoint policies can apply Classifications to files uploaded to Sharepoint in the form of metadata tags. Once Classifications are applied, you can design Access Control policies that govern how the documents are access, used, and distributed. Documents can be tagged individually upon upload based on content that is present in the file (content aware automated tagging), file properties, file type, identity of the user uploading the document, and more. Existing documents already present in SharePoint can also be classified using an automatic batch content analysis processes.

Access and Usage Control Policies

Access Control policies in SharePoint come in various forms. Most simply, policies can allow or block access to a document. For Access Control policies, you can define a SharePoint portal as a file location: the various sites, libraries, document lists, etc., are analogous to directories, whose content you want to control in certain ways. Access Control policies either allow access to a portal location, or denying it, for either reading or writing purposes, or both.

Access Control policies can also control the creation of SharePoint webparts (lists, libraries and so on). This functionality provides a solution to one of the most difficult control problems of SharePoint: limiting how individual users can share data resources by opening new lists, to which they have control over access by others. You can limit or monitor the creation of the following kinds of SharePoint structures: Document, Libraries, Announcements, Project Tasks, Form Libraries, Contacts, Issue Tracking, Wiki Page Libraries, Discussion Boards, Surveys, Picture Libraries, Links, Custom Lists, Slide Libraries, Calendars, Custom List in Datasheet Views, Translation Management Libraries, Tasks, Languages and Translators, and Imported Spreadsheets.

Access Control policies can also limit the ways users access and modify site permissions.

In addition to this form of Access Control, you can use the Entitlement Manager for Security trimming, which controls what information SharePoint users are able to view (not access). There are three kinds of Security Trimming: Webpart trimming controls which webparts display when a user views sites in SharePoint, List Trimming controls which items in a list display, and Search Trimming determines which search results display when a user runs a search within SharePoint.

Data Protection Policies

Classifications can also be applied to documents as they are downloaded out of SharePoint. This enables data to remain protected in accordance with an organization’s policies even when it leaves the SharePoint server, for example, how data is printed, distributed, and where it is stored. Just as with classification on upload, classification on download can be applied in several ways: through user-selected tags, SharePoint column tags, and policy designer defined tags.

Auditing Policies

You can design SharePoint policies that automatically log specific kinds of user events so that they can be audited.
