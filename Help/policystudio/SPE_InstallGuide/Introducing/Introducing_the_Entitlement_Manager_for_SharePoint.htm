Introducing the Entitlement Manager for SharePoint

Microsoft SharePoint has become an extremely widespread tool in the enterprise network environment, largely due to its flexibility and versatility in helping multiple users collaborate on projects. Unfortunately, the ease with which it allows collaborators to share data and manage their own fine-tuned access controls has created serious risks in terms of inappropriate data access, distribution, and loss.

To address this challenge, the Entitlement Manager for SharePoint is designed to enforce policies that control whether and how users can access, download, and use data stored on a SharePoint server. SharePoint policies can apply to entire portals or to any parts of them, including sites, pages, or webparts like links, lists, or calendars. Policies can allow some users to view all webparts on a page (document lists, for example), while blocking other users from viewing some subset of the webparts on the same page. Policy can also block classes of users from viewing results returned by SharePoint’s own content search tool. The Entitlement Manager for SharePoint offers much more granular control than the internal Access Control List (ACL) tools available within SharePoint itself, and its enforcement decisions override the page-or site-level permissions granted by page or site owners.SharePoint policies can control user activity in both directions—accessing and downloading, and posting or uploading. This granularity of control allows employees to collaborate, while still maintaining compliance with organizational polices regarding how sensitive data and intellectual property should be used, viewed, and distributed.
