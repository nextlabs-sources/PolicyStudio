Logical Architecture

The figure below represents the logical architecture of the Entitlement Manager for SharePoint. It consists of two basic parts: a generic Policy Controller, and a Policy Adapter that is specific to Entitlement Manager for SharePoint. This logical architecture is common to all NextLabs Entitlement Managers.

The Policy Controller provides the interface to the Policy Server, and serves as the Policy Decision Point, or PDP. It comprises a set of software modules that are delivered ready-to-install on the Entitlement Manager hosts. Because it is not specific to any Entitlement Manager type, it requires no customization.

 

Logical Architecture

The Policy Adapter provides the interface to whatever specific application or system you want to enforce, and serves as the Policy Enforcement Point, or PEP. Each of the available pre-built Entitlement Manager includes a different policy adapter, specific to the application or system it enforces. If you need a new adaptor for some other system—your proprietary CRM application, for example—this is the custom piece you need to create using the Policy Server Libraries.

Note that the each piece has its own separate installation process, and the Policy Controller should be installed first.

The following sections present more detail on the functional components that make up the Policy Controller and the Policy Adapter. These do not necessarily map to separate software processes, but rather are logical components representing discrete functions of the two parts. This is helpful in understanding the various contributions each part makes to the Entitlement Manager’s overall operation.
