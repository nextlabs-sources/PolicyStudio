The Policy Adapter

The SharePoint Policy Adapter tightly integrates with the Microsoft SharePoint event and object model to monitor application activities and enforce policy decisions for information controls. It consists of the following logical parts:

1       The Protocol Monitor monitors HTTP requests with ASP.Net’s HTTP Mod­ule and blocks requests according to policy decisions. It provides one of the Entitlement Manager’s two PEPs.

2       The Application Monitor monitors SharePoint events by integrating with SharePoint application, and blocks event execution according to policy decisions. It provides the system’s other PEP.

3       The Context Manager retrieves SharePoint object properties and pro­vides application context for monitored events.

4       The Notification Manager triggers SharePoint to display an educational message to inform a user of policy and procedures, or block the user event and inform the user about the violation.

5       The Policy Controller Client sends policy decision requests to the Policy Controller, along with event details and application context.

Note that the first four components—all but the Policy Controller Client—work together as the Entitlement Manager’s Policy Enforcement Point, or PEP.
