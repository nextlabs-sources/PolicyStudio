Setting Up the Module

The SIG Reporting Module is delivered together with every SharePoint Server Enforcer. You can add it to an already running Control Center, or set it up as part of an initial platform installation after the standard installation is complete. Setting up the module involves two steps:

1       Activating the pre-defined reports

2       Importing the custom obligations

Activating the Predefined Reports

The predefined SIG reports are included in every standard installation of Reporter 5.5; you simply need to activate them. You do this by the following procedure:

  1. Using any text editor, open the Control Center configuration file configu­ration.xml. By default, this file is located at

    2. c:\Program Files\NextLabs\Policy Server\server\ configuration\configuration.xml

  2. Search for the <Reporter> section, as shown below.

    3.  

    Activating the SIG Reports Tab

  3. The default value for the <ShowSharePointReports> element in this sec­tion, “0”, hides the special Reporter tab that displays all predefined SIG reports. To display the reports, change this value to “1” and save your change.

  4. Restart your Policy Server, and the new SIG tab with all predefined reports will display when you open Reporter.

Importing the Custom Obligations

Importing the custom obligations requires manually editing the system configuration file, configuration.xml. Specifically, you need to add two obligation sections into the system configuration file. When you save your changes and restart your Policy Server, these obligations will be available for use in the Custom Obligations combo-box of any new policy you create in Policy Server, as shown below.

 

SIG Custom Obligations

Follow these steps:

  1. Use Wordpad to open your Control Center configuration file, configura­tion.xml.

  2. Search for the <Obligations> section of the configuration file.

  3. Using any text editor, open the SIG configuration snippet file, SIG_Reporting_Module_Obligations.xml, which you received as a part of the module.

  4. Copy the two obligation sections from the snippet file into the <Obliga­tions> section of your live configuration file. An example of one of these obligations is shown in the figure below.

  5. Close your live configuration file, saving your change.

  6. Restart your Policy Server, to force the system to read the new version of the configuration file.

  7. Start Policy Studio, and check that the obligations are available by creat­ing a new policy and pulling down the Custom Obligations combo-box. They should display in the list of custom obligations there.
