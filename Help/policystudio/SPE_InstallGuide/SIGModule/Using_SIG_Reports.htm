Using SIG Reports

This section explains how you use the set of predefined reports that is included in the SharePoint Information Governance (SIG) Reporting module.

The SIG Tab

Once you have installed the SIG Reporting module, Reporter displays a third tab labeled SharePoint Information Governance. All predefined SIG reports are displayed on this tab, in the upper left-hand pane. These reports are specially designed to help managers monitor and understand how users in the enterprise are accessing and using SharePoint resources. This knowledge is very helpful in designing SharePoint access policies, and even in designing and modifying the structure of the SharePoint portal itself.

You use these reports the same way you would your own custom defined reports on the My Reports tab: open a report, select filtering criteria, and run the report. Unlike the contents of the My Reports tab, however, you cannot add new reports to this tab, or edit the predefined reports.

Lifecycle Reports

The data in SharePoint Lifecycle Reports is gathered and stored whenever a user performs some action that is covered by a currently active SharePoint Control Policy—basically, any policy that includes the Log SharePoint Control obligation in it.

These reports focus on the volume of SharePoint activity—that is, frequency of users performing various actions on various SharePoint resources. All lifecycle reports display SharePoint activity grouped by one entity, such as individual user or department. That is, each bar in the chart represents one of whatever entity the chart is grouped by: one user, one department, one business process, and so on; and the height of the bar indicates how many times users in that have accessed or used SharePoint resources, in the course of the defined report period. Note that this is an aggregate count, which includes all actions performed on all resources in the SharePoint portal, consistent with active SharePoint Control policies.

In addition, each activity group (that is, bar) is further segmented based on the lifecycle stage of the data in use. Each color coded segment on each bar represents the fraction of total activity that concerned data in each of four possible lifecycle state: Modification, Consumption, Storage, or Deletion. This allows auditors or analysts to understand at a glance the lifecycle stages at which data is of greatest and least interest to specific groups of users in the enterprise.

The Four Lifecycle Stages

All of the module’s predefined lifecycle reports break down SharePoint activity into four lifecycle stages—a standard information governance concept. Each stage maps to one or more actions a user might perform on a SharePoint resource; the following table represents this mapping.

Lifecycle Stage

Covers SharePoint Action(s)

Modification

Create, Edit, Rename

Consumption

Open, Print

Storage

Export, Copy, Move

Deletion

Delete

As the table shows, this is a one-to-many mapping. Thus, for example, the Consumption (purple) segments in the sample chart shown below represent all instances when members of that department either opened or printed a SharePoint resource. These actions are aggregated; the report does not break down how many of the consumption events were Open and how many were Print.

Policy Activity Reports

The data in SharePoint Policy Activity Reports is gathered and stored whenever a user performs some action that is covered by a currently active SharePoint Activity Logging policy—basically, any policy that includes the Log SharePoint Activity obligation in it.

This group of reports displays aggregate SharePoint activity, not broken down by lifecycle, but represented from six different perspectives—that is, filtered six ways:

3       By Department

4       By SharePoint Site

5       By Business Process

6       By Individual User

7       By Policy

8       As a Trend, over time

Generating SIG Reports

As we have seen, each kind of report displays data gathered in the course of enforcing SIG Reporting Policies. The two types of such policies map directly to the two types of reports:

9       SharePoint Control Policies provide the data content for Lifecycle reports

10   SharePoint Activity Logging Policies provide the data content for Activity reports

You must construct and deploy the appropriate policies and give them some time to collect activity data before you can generate any meaningful reports.

To generate a report,

1.Select it from the list on the SIG tab.

2.In the From and To fields, define the period you want the report to cover.

3.If the report you are running has one or more special filter fields (see Special Filter Fields), provide a value for each that suits your needs.

4.Click the Run button.
