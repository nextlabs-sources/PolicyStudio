About Content Aware Classification

You can write policies that scan the content of SharePoint documents and automatically tag them when specific keywords or content types are detected. Note that the only kinds of policies that can apply content analysis for the Entitlement Manager for SharePoint are classification policies (in other words, you cannot allow or block access based on content, although you can allow or block access based on classification values once they are present).

Content Aware Classification for SharePoint works by applying a value to a pre-existing column in a document library or list. You can apply content analysis to SharePoint documents either upon the upload of the new document, or using a batch process that will automatically classify all existing documents in designated lists or libraries.

Example Policy:  Content Aware Classification on Upload  

After you have performed the preliminary SharePoint setup and planning, you are ready to create content aware policies. Let’s take the example use case that anytime an engineer uploads a new document to a designated site, we want to perform content analysis on the document to as certain whether it should be classified as “intellectual property” or not. We design a content analysis policy that searches for specific key terms. We would perform the following steps:

  1. Create a Document policy to Monitor the Action of Uploading documents.

    2. Note: Policies that apply content analysis in SharePoint must be “Monitor” policies for the “Upload” action. You cannot create Allow/Deny poli­cies that apply content analysis, or policies that use a different Action (such as Create/Edit).

  2. Create a Resource Portal Component for the site to be targeted in policy. We create one for our engineering site (“Engineering Site Collection” in the example below).

    3.  

    Creating a Policy for Content Analysis upon Upload

  3. Create another Resource Portal Component for content analysis (see below).

    4. 20   In the “With Content” area, select a Property Name from the drop down menu. Select “Contains Content” to analyze content, or select “Content Type” to search content of a particular format or type.

    21   The contents of the next drop down list depend on what was selected in the first. Since we selected “Contains Content,” we can select a “Key­word” in the second drop down list.

    22   In “Contains Match Count Greater or Equal To,” you can specify the number of incidences upon which the policy should be triggered. For example, if you want the policy to be triggered on a single instance of a keyword, select “1.”

  4. Select the Custom Obligation "Set Column Value." Enter the target Column Name, and the Column Value that should be applied if the policy evaluates to true.

After this policy is saved and deployed, anytime a user uploads a document to the engineering site collection, the document will be checked for the keyword “Product Name.” If it is present three or more times, the value to the IP column will automatically be set to “Yes.”

Applying Content Analysis to Existing Documents as a Batch Process

In addition to applying content analysis to SharePoint documents based on upload events, you can run batch processes on a designated library or list. When the batch process runs, it automatically analyzes content and applies appropriate tags, based on the Content Aware classification policies deployed to the Entitlement Manager (discussed in the previous section). (In our example use case, the batch analysis will apply to the Engineering Site Collection and scan for the presence of keyword “Product Name.”)

Note: To start a content analysis batch process, you must log in as non-ADFS user. A user logged in via ADFS is not permitted to start or schedule a content analysis batch process.

To schedule a content analysis batch process:

  1. In the target library or list, go to Settings > Document Library Settings.

    2.  

    Document Library Settings

  2. In Permissions and Management, click Analyze Content.

    3.  

    Clicking Analyze Content

  3. In the Content Analysis window, you can create schedules for content analysis that will be performed on the library or list.

8       To create a new schedule, click Create Schedule. A window displays where you specify how frequently content analysis should be per­formed and what time of the day the process should start.

9       To apply an already defined schedule, select it from the drop down menu, and click Start Content Analysis.
